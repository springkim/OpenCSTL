Quick Comparison

C++ STL vs OpenCSTL

Drop-in familiar API — no macros, no forward declarations

C++ STL
std::vector<int> v;
v.push_back(42);
v.push_back(7);
for (auto x : v) {
    printf("%d\n", x);
}
v.clear();

std::map<int,int> m;
m.insert({1, 100});
OpenCSTL
VECTOR(int) v = cstl_vector(int);
cstl_push_back(v, 42);
cstl_push_back(v, 7);
for (int *it = cstl_begin(v); it != cstl_end(v); ++it)
    printf("%d\n", *it);
cstl_clear(v);

MAP(int) m = cstl_map(int, int, IntCmp);
cstl_insert(m, 1, 100);
Why OpenCSTL

The C Container Library
Done Right

Everything you need, nothing you don't.

📦

Single Header

One #include is all it takes. No build system configuration, no CMake integration, no linking steps.

⚡

Identical STL Interface

The same function names as C++ STL — push_back, insert, find, begin, end. No new API to learn.

🔢

Bracket Access

vector and deque support v[i] subscript access, identical to C arrays and C++ STL.

🔀

Auto Dispatch

Functions automatically route to the correct implementation based on container type — like C++ operator overloading.

🌍

Cross-platform

Windows (MSVC, MinGW64), Linux (GCC), and macOS (Clang). All major compilers, all major platforms.

🔁

Full Iterator Support

All containers expose begin/end/rbegin/rend iterators. Works with qsort and bsearch.

🚫

Zero Boilerplate

No forward declarations, no type registration macros, no initialization ceremony. Declare and use immediately.

🎯

Variadic API

Parameter-count based dispatch mirrors C++ overloading — resize(n) and resize(n, val) both work naturally.

Reference

Containers

All 9 STL-compatible containers, fully implemented.

cstl_vector Dynamic Array

Contiguous storage with O(1) random access and amortized O(1) append.

 cstl_list Doubly Linked List

O(1) insert/erase anywhere with bidirectional iteration.

 cstl_deque Double-Ended Queue

O(1) push/pop at both ends with random access support.

 cstl_stack LIFO Adapter

Last-in first-out container adapter built on deque.

 cstl_queue FIFO Adapter

First-in first-out container with priority_queue variant.

 cstl_set Red-Black Tree

Sorted unique keys with O(log n) lookup, insert, erase.

 cstl_map Red-Black Tree

Sorted key-value pairs, same O(log n) guarantees as set.

 cstl_unordered_set Hash Table

Unordered unique keys with O(1) average-case operations.

 cstl_unordered_map Hash Table

Unordered key-value store with O(1) average-case operations.
