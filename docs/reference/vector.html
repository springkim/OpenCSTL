cstl_vector

Dynamic contiguous array with O(1) amortized append and subscript access. Equivalent to std::vector<T>.

SequenceRandom AccessContiguousvector.h

Declaration

without USE_CSTL_FUNC
VECTOR(int) v = cstl_vector(int);
// Subscript access
v[i]
with USE_CSTL_FUNC
#define USE_CSTL_FUNC
VECTOR(int) v = new_vector(int);
// Subscript access (same)
v[i]
ℹ

VECTOR(TYPE) expands to TYPE*. The header metadata is stored just before the data pointer, so v[i] is a true pointer dereference — zero overhead. The buffer is fully compatible with qsort, bsearch, memcpy, and fwrite.

Example

c
#define USE_CSTL_FUNC
#include "opencstl.h"

void example_vector(void) {
    VECTOR(int) arr = new_vector(int);

    for (int i = 0; i < 10; i++) {
        push_back(arr, i);
    }
    // iterator loop
    for (int *it = begin(arr); it != end(arr); it++) {
        printf("[%3d]", *it);
    }
    puts("");

    // subscript access
    printf("arr[3] = %d\n", arr[3]);   // 3

    // sort with standard qsort
    qsort(begin(arr), size(arr), sizeof(int), IntCmp);

    destroy(arr);
}

Notes

⚠

Any operation that increases size beyond capacity (push_back, insert, resize) may reallocate the internal buffer. Do not cache raw element pointers across such calls — they may be invalidated.

✓

Data is contiguous in memory. You can pass begin(v) directly to qsort, bsearch, fwrite, or any C API expecting a plain array.
