cstl_map

Sorted key-value store backed by a Red-Black tree. Equivalent to std::map<K,V>.

AssociativeSortedBidirectional Iteratortree.h

Declaration

without USE_CSTL_FUNC
// MAP(KEY) m = cstl_map(KEY, VALUE, cmp)
MAP(int) m = cstl_map(int, int, COMPARE(int));

// KEY can be any type, including char*
MAP(int) d = cstl_map(int, char*, COMPARE(int));
with USE_CSTL_FUNC
#define USE_CSTL_FUNC
MAP(int) m = new_map(int, int, COMPARE(int));

MAP(int) d = new_map(int, char*, COMPARE(int));
ℹ

The iterator (KEY_TYPE *it) points to the key. Access the associated value with cstl_value(it, VALUE_TYPE). Internally each node stores [key | value] contiguously, so cstl_value is simply an offset dereference — zero overhead.

Example

c
#define USE_CSTL_FUNC
#include "opencstl.h"

void example_map(void) {
    char *containers[] = {
        "vector", "list", "set", "map", "stack",
        "queue", "priority_queue", "unordered_map", "unordered_set"
    };
    size_t len = sizeof(containers) / sizeof(containers[0]);
    printf("Number of elements: %zu\n", len);

    // KEY=int, VALUE=char*
    MAP(int) d = new_map(int, char*, COMPARE(int));
    for (int i = 0; i < len; i++) {
        insert(d, i, containers[i]);
    }

    // reverse iteration — *it = key, cstl_value(it, char*) = value
    for (int *it = rbegin(d); it != rend(d); it = prev(it)) {
        printf("[%3d : %s]\n", *it, cstl_value(it, char*));
    }
    puts("");

    destroy(d);
}

Notes

✓

cstl_value(it, VALUE_TYPE) is a macro that computes the value address directly from the iterator pointer — it does not do a tree lookup. It is O(1) and safe to use in tight loops.

💡

Use cstl_unordered_map when you don't need sorted order and want O(1) average-case operations instead of O(log n).
